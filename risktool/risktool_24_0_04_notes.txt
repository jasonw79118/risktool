RISKTOOL 24.0.04 NOTES

Purpose: stabilization patch only. New feature work is paused until persistence and dashboard navigation are stable.

Changed:
- Restored scenario save persistence so edits are written to the active saved scenario record.
- Save now stores the full scenario payload, not just a stripped summary row.
- Removed the workspace-folder gating effect from scenario save/list rendering. Reconnect through User Settings remains available, but saving no longer silently fails because a folder is not connected.
- Added best-effort workspace JSON copy only when a workspace folder is already connected.
- Reconnected Dashboard Open Report buttons.
- Added a delegated click handler so Open Report still works after dashboard table rerenders.
- Locked runtime/version display to 24.0.04.

Not changed:
- Portfolio Report expansion was not changed in this patch.
- Test Mode and Model Validation were not changed in this patch.
- Post-login load workspace flow remains ignored/de-scoped.
- Existing workspace architecture was not replaced.

Regression test:
1. Open an existing scenario, change a field, click Save Scenario, navigate away and reopen it. The change should persist.
2. Create a new Single scenario and save it. It should appear in Saved Scenarios and Dashboard Open Scenarios.
3. Create or edit a Complex scenario and save it. It should appear in Saved Scenarios and Dashboard Open Scenarios.
4. From Dashboard, click Open Report for an open scenario. It should navigate to the report view for that scenario.
5. Refresh the browser and confirm the saved scenario list is still present.
6. Confirm bottom-left version stays 24.0.04.
