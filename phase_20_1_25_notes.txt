RISKTOOL HANDOFF DOCUMENT

PROJECT OVERVIEW

Risktool is a web-based enterprise risk management application hosted on
GitHub Pages. It follows a phased release model with version tracking
(e.g., 20.1.xx).

PRIMARY OBJECTIVE

Build a Hubbard-aligned risk modeling tool (without explicitly
referencing Hubbard) that supports: - Single Scenario risk analysis -
Complex Scenario (multi-component) analysis - Monte Carlo simulation
outputs - Insurance effectiveness evaluation - Evidence-based risk
inputs - Board-ready reporting

CRITICAL DEVELOPMENT RULES

1.  NEVER paste code into chat
    -   ALL code must be delivered as ZIP files
    -   Files must retain exact filenames (app.js, index.html, etc.)
2.  ALWAYS include index.html updates
    -   Version number must increment every phase
    -   Dashboard version display is required
3.  DO NOT break existing functionality
    -   No global listeners (caused freezes)
    -   No MutationObservers
    -   No page-wide hooks
    -   Changes must be surgical
4.  UI STRUCTURE RULE
    -   Vertical layout ONLY
    -   Section → Table → Section flow
    -   NO side-by-side sections

INFORMATION (USER MANUAL) SYSTEM

-   Must live ONLY in left navigation “Information” menu
-   No floating buttons
-   No duplicate entry points

Current State: - Manual is now rendering correctly - Styled with
enterprise UI (pill navigation, cards, spacing) - Sections include: -
Getting Started - Single Scenario Guide - Complex Scenario Guide - Field
Guidance - Results Interpretation - Insurance Evaluation - Evidence
Usage - FAQ - Reference

NEXT PHASE DIRECTION

Move from static manual → interactive help system:

1.  Field-level hover help
2.  Context-aware guidance (based on screen)
3.  Tooltips on inputs
4.  Expand/collapse manual sections
5.  Link manual sections to actual fields

RISK AREAS (DO NOT REPEAT)

-   Phase 20.1.16 caused full app freeze due to:
    -   global click listeners
    -   MutationObserver

Lesson: ONLY integrate into EXISTING flows, never override them
globally.
